2. Setup paso a paso

2.1 Instalar Claude Code CLI

npm install -g @anthropic-ai/claude-code
claude --version   # Debe mostrar un numero de version

La primera vez que ejecutes claude te pedira la API key de Anthropic.

2.2 Clonar repositorios

# Repo principal (donde trabajaras)
git clone https://github.com/LenderoCapital/lenderohub-playground.git

# Repo de referencia (solo lectura, para ejercicio avanzado L4.2)
git clone https://github.com/LenderoCapital/lenderomo-playground.git
Si git clone falla con "repository not found"

Aun no tienes acceso. Envia tu username de GitHub al equipo y espera la invitacion.

2.3 Setup Backend

cd lenderohub-playground/lenderohub-backend
cp .env.local.example .env.local
npm install
npm run seed:playground

El seed crea 3 usuarios de prueba en la base de datos local.

2.4 Setup Frontend

cd ../lenderohub-frontend
cp .env.local.example .env.local
npm install

2.5 MongoDB con Docker (si no lo instalaste directamente)

docker run -d -p 27017:27017 --name mongodb mongo:7

Despues del seed, verifica: docker exec mongodb mongosh --eval "db.getSiblingDB('lenderohub-playground').users.countDocuments()" — debe devolver 3.

3. Verificar que todo funciona

Abre 2 terminales y ejecuta:

Terminal 1 — Backend
cd lenderohub-backend
npm run dev

Debe decir "Servidor listo" en puerto 3000

Terminal 2 — Frontend
cd lenderohub-frontend
npm run dev

Debe decir "Ready" en puerto 3001

Checklist de verificacion

  • http://localhost:3000/api/health responde OK
  • http://localhost:3001 carga la pagina de login
  • Login con admin@playground.local / playground123 funciona
  • claude --version responde
  • Cursor IDE abre el proyecto sin errores
Si todo esta bien

Ya puedes empezar con los ejercicios. Abre playground/level-0/01-environment-setup.md

5. Herramientas AI

Usaras dos herramientas AI complementarias. Cada una es mejor para tareas distintas:

Claude Code
AI en la terminal que lee, escribe y ejecuta
Mejor para:
  • Explorar el codebase completo
  • Crear features multi-archivo
  • Debugging (lee logs, ejecuta tests)
  • Git operations (commits, branches)
  • Code review
Cursor
IDE con AI integrado (fork de VS Code)
Mejor para:
  • Editar codigo inline (Cmd+K)
  • Autocompletado en tiempo real (Tab)
  • Preguntas sobre el archivo abierto
  • Cambios pequenos y precisos
  • Navegacion visual del codigo
Regla de oro

No copies y pegues codigo de AI sin entenderlo. Lee cada linea. Si algo no tiene sentido, preguntale al AI por que lo hizo asi. Aprende de los errores del AI tanto como de los aciertos.

6. Mapa de ejercicios

19 ejercicios organizados en 6 niveles. Tiempo estimado total: 80-105 horas.

Level 0
Foundations
6-8 horas

Setup, exploracion y primeros pasos con herramientas AI.

  • 0.1 Environment Setup — instalar todo y verificar
  • 0.2 First Run — explorar la app corriendo, DevTools, curl
  • 0.3 Codebase Tour — mapear la arquitectura backend + frontend
  • 0.4 AI Tool Comparison — Claude Code vs Cursor, cuando usar cada uno
Level 1
Frontend Basics
12-16 horas

Componentes React, tablas, formularios y llamadas API.

  • 1.1 Static Alert Card — crear un componente de alerta reutilizable
  • 1.2 Data Table — tabla con sort, filter y paginacion
  • 1.3 Form Validation — formulario con validacion Zod
  • 1.4 API Call Display — consumir API mock, loading/error states
Level 2
Backend Basics
12-16 horas

Endpoints, modelos MongoDB, CRUD y middleware de permisos.

  • 2.1 Health Endpoint — crear endpoint detallado de health
  • 2.2 Mongoose Model — definir un modelo con validacion y indices
  • 2.3 CRUD Controller — controller completo con paginacion
  • 2.4 Permissions Middleware — middleware de autorizacion por rol
Level 3
Full-Stack Features
15-20 horas

Features completas que conectan frontend + backend + base de datos.

  • 3.1 Notifications UI + Backend — sistema de notificaciones end-to-end
  • 3.2 Activity Log — log de actividad con filtros y timeline
  • 3.3 Dashboard Widget — widget con datos en tiempo real
Level 4
Advanced Patterns
20-25 horas

Patrones avanzados de MongoDB, RBAC y operaciones masivas.

  • 4.1 Discriminator Models — herencia de modelos en Mongoose
  • 4.2 RBAC Matrix — sistema de permisos por rol/etapa/accion (referencia: LenderoMO)
  • 4.3 Bulk Operations — importacion/exportacion masiva con validacion
Level 5
Capstone
15-20 horas

Feature completa desde cero. Demuestra todo lo aprendido.

  • 5.1 Announcement Board — sistema completo de anuncios (model, API, UI, permisos, notificaciones)

7. Como trabajar cada ejercicio

Flujo para cada ejercicio:

  1. Lee el archivo del ejercicio completo antes de escribir codigo
  2. Lee los "Archivos de Referencia" — son codigo existente que sirve como ejemplo
  3. Planea antes de implementar (usa Claude Code para explorar)
  4. Implementa paso a paso siguiendo las instrucciones
  5. Verifica cada criterio de aceptacion al terminar
  6. Marca tu progreso en playground/PROGRESS.md
Usa herramientas AI inteligentemente

Cada ejercicio tiene una seccion "Guia de AI Tool" con sugerencias especificas de cuando usar Claude Code y cuando usar Cursor. Siguelas.

Reglas

  • Levels 0-2 son secuenciales — cada uno construye sobre el anterior
  • Levels 3-5 pueden hacerse en cualquier orden dentro del nivel
  • No avances sin completar todos los criterios de aceptacion
  • Siempre ejecuta npm run typecheck despues de cada cambio para verificar tipos

8. Tips y mejores practicas

Sobre el codigo

  • TypeScript es tu amigo. Si algo esta en rojo, arreglalo antes de continuar. npm run typecheck debe pasar siempre.
  • Sigue los patrones existentes. Antes de crear algo nuevo, busca como se hace en el codebase actual.
  • Un cambio a la vez. Haz commits frecuentes. Si algo se rompe, puedes volver atras.
  • Lee antes de escribir. Los mejores developers pasan mas tiempo leyendo codigo que escribiendolo.

Sobre herramientas AI

  • AI genera el 80%, tu escribes el 20% critico. Usa AI para boilerplate, escribe la logica de negocio tu.
  • Siempre verifica el output del AI. No todo lo que genera es correcto.
  • Aprende a escribir buenos prompts. "Crea un componente" es peor que "crea un componente de tabla con sort y filter siguiendo el patron de beneficiarios/page.tsx".
  • Cuando el AI se equivoca, explicale por que. Asi mejora el siguiente intento.

Sobre productividad

  • DevTools es tu mejor herramienta de debugging. Network tab, Console, Elements.
  • Usa el terminal. curl para probar endpoints. mongosh para inspeccionar datos.
  • Documenta lo que aprendes. Usa la seccion "Notas personales" de PROGRESS.md.
